视界问题

粒子视界

我们有一个问题，这个问题就是我们当前可以观测到的宇宙有多大？可以想象宇宙最初的哪一个，某一个位置A发出一个光速运行的粒子，如果这个粒子现在正好到达我们地球，那么我们现在正好看到A地的情况，只不过是宇宙最初那一个刻的。此时，A地距离我们的距离，这就是我们能够看到的宇宙的最远处。这个距离又称为粒子视界。这个距离是
$$
d_p= \int_0^{t_0} \frac{a(t_0)}{a(t)} c dt
$$
其中$d_p$是粒子视界，$t_0$是我们现在的时间，也就是138亿年，$c$是光速。$a(t_0)$是现在宇宙的大小，$a(t)$是$t$时刻的宇宙的大小。这个公式其实是很平庸的，你知道对光从初始时刻到现在飞过的距离积分就可以了，只是你要知道光在$t$时刻飞过的距离$c dt$，现在由于宇宙膨胀变为了$\frac{a(t_0)}{a(t)} c dt$。

根据我们现在我们宇宙的膨胀模型，大约$70%$的暗能量，$30%$的物质控制我们的宇宙的膨胀。据此我们可以估计过去每一刻宇宙的大小。然后把$a(t)$带入上式（技术上写为对$d a$的积分更加方便， 但是读者可以不理会这个技术细节）就可以得到现在的视界大小是
$$
d_p=469亿光年
$$
这个数值要比现在的宇宙的年龄乘以光速要大，这并不奇怪，虽然光飞行了138亿年，但是宇宙在膨胀，过去飞过的一光年，演化大小到现在，要大于一光年。

这是我们现在能够观察到宇宙的最远处，那么我们实际上能够观察到这么远的距离吗。答案是是。之所以说是，我们我们现在观察到的最遥远的光子，宇宙微波背景辐射，来自于宇宙年龄为38万年时候的热辐射。38万年，相比于宇宙138亿年的历史太短暂了，可以说这些光子从某个点经过138亿年飞行来到我们地球，而这个点现在距离我们469光年。

事件视界

我们假设在遥远的远方此刻发生了一个事件，那么我们能在未来某一个时刻观察到这个事件吗。如果这个事件是一个发光的过程，那么我们问题就是，发出的光信号能达到地球吗？如果不考虑宇宙的膨胀，那么答案当然是是。只要有足够的时间，那么光信号迟早能够传播到地球。但是宇宙是不停膨胀的，宇宙膨胀的足够快，那么即使到了永远，我们也可能观察不到这个事件。到了永远，光信号能够传达到A，A点到事件发生的原初点距离是
$$
de= \int{t_0}^{\infty} \frac{a(t_0)}{a(t)} c dt
$$
你需要注意到的是，光在t（$>t_0$）时刻飞过的距离$c dt$是在未来膨胀后的宇宙飞行的，在现在其距离只有$\frac{a(t_0)}{a(t)} c dt$。如果现在地球距离这个事件距离小于$d_e$，那么这个事件迟早会被地球上的我们在未来某一刻观察到。反之，如果现在地球距离这个事件距离大于$d_e$，那么这个事件永远不能被地球上的我们的某一刻观察到。

按照现在的宇宙模型计算
$$
d_e=160亿光年
$$
也就是说，如果此刻发生一个事件，只有这个事件的距离小于160亿光年，我们在才能在未来某个时刻观察到这个事件。

视界的和

那么最遥远的未来，我能够观察到最远的事件，距离我们能有多远呢？为了能够观察到最远的事件，这个事件必须是宇宙一开始就发生的，这个距离是

$$
d = \int_{0}^{\infty} \frac{a(t_0)}{a(t)} c dt
$$

这个距离的大小正是
$$
d = 629 亿光年
$$
也就是说在这个距离之外的位置上发生的事件，无论什么时间发生的，都不可能影响到我们，我们也不可能观察到事件的发生，即使这个事件是宇宙一开始时发生的。

这是悲剧的，这表示我们能够观察到宇宙大小是有限的。按照现在的观测，我们的宇宙是无限大的，所以从概率上说我们人类不应该是宇宙的孤儿。但是人类能够观察到宇宙大小是有限，地球可能是我们视界之内唯一有生命的星球。如果因为视界问题，我们不能观察到外星生命，同样外星生命也不能意识到我们的存在。

“如果存在外星人类呢？”
“外星人类也是孤单的！”

博彩


我们研究一下博彩问题。

假设有两支队伍比赛，比如说A队和B队，投注者可以投注其中一只队伍。无论投注者所投的队伍输赢，投注者都要首先付出一定的本金。如果投注者投注的队伍赢了，投注者这可以获取一定的回报；如果投注者投注的队伍输了，那么投注者没有回报。每投入一元的回报就是所谓的赔率。一般两队的赔率并不相同，赢面比较大的队伍赔率比较低，而且赢面比较小的队伍赔率比较低。所谓低风险低收入，高风险高收入。赔率都是大于1的，否则回报不抵成本。

博彩公司的策略

对于博彩公司而言，首要的是增加收益，降低风险。

最小化风险

在我们的研究里，假设降低风险是首要的。假设买A队赢的资金为 $N_A$ ，买B队赢的资金为 $N_B$ 。那么如果A队赢了，那么博彩公司正收益的条件为
$$
N_A O_A <= (N_A + N_B)
$$
或者说
$$
N_A / (N_A + N_B) <= 1/ O_A
$$
那么如果B队赢了，那么博彩公司正收益的条件为
$$
N_B O_B <= (N_A + N_B)
$$
或者说
$$
1 - 1/ O_B <= N_A / (N_A + N_B)
$$
联合两个公式可得
$$
1 - 1/ O_B <= N_A / (N_A + N_B) <= 1 / O_A
$$
如果以降低风险为首要条件，那么博彩公司首先应该估计$N_A / (N_A + N_B)$，然后定下赔率$O_B$和$O_A$。如果投注者实际的$N_A / (N_A + N_B)$与估计$N_A / (N_A + N_B)$的比较接近，都在$[1 - 1/ O_B, 1 / O_A]$区间内，那么博彩公司实际上是稳赚不赔的。如果$N_A / (N_A + N_B)$不在$[1 - 1/ O_B, 1 / O_A]$区间内，那么博彩公司要么大赚，要么大赔，这取决于到底那支队伍赢。博彩公司可以根据投注情况实时更改赔率，即由实时的$N_A / (N_A + N_B)$来调整$O_B$和$O_A$。

对于博彩公司，赔率不应设置的过高，否则区间过小，增大了出现风险的可能性；同时赔率不应设置的过低，否则对于投注者来说投注就不再有吸引力了。

我们从上式可以看出$1 - 1/ O_B <= 1 / O_A$，即$1/O_A + 1/O_B>1$。实际的博彩设置都是满足这个关系式的。

这里可以看到，赔率是通过投注者的投注者行为确定的。那么投注者的策略又是什么？

投注者的策略

我们知道那只队伍输赢有一定的或然性。假设队伍A赢得的概率是$P_A$，队伍B赢得的概率是$P_B$。那么投注A队伍的收益期望为$P_A O_A - 1$。如果$P_A > 1 / O_A$，那么就可以投注A队伍。投注B队伍的收益期望为$(1 - P_A) O_B - 1$。如果$P_A < 1 - 1 / O_B$，那么就可以投注B队伍。或者说如果$P_A$不在区间$[1 - 1/ O_B, 1 / O_A]$，那么就可以通过投注某一支队伍而获得正的收益期望。什么时候会出现这种投注者的机遇呢？

最大化收益

我们假设博彩公司是最小化风险的，而投注者是最大化收益的。我们知道$N_A / (N_A + N_B)$在区间$[1 - 1/ O_B, 1 / O_A]$里，同时要求$P_A$不在区间$[1 - 1/ O_B, 1 / O_A]$里。这要求$N_A / (N_A + N_B)$与$P_A$差别足够大，并且区间$[1 - 1/ O_B, 1 / O_A]$足够小。

区间$[1 - 1/ O_B, 1 / O_A]$足够小，意味着博彩公司克制的抽成，这很容易理解，一般这个数值在10%左右。

为了简单起见，我们假定$[1 - 1/ O_B, 1 / O_A]$足够小，那么 $1 - 1/ O_B = 1 / O_A = N_A / (N_A + N_B)$。我们知道博彩公司首先会设置一个初始赔率，然后根据投注情况设置更改赔率，这叫做市场赔率。那么$P_A$与$N_A / (N_A + N_B)$是什么关系呢？如果$P_A>N_A / (N_A + N_B)$那么投注A队是有有收益的，所以$N_A$就会增加，直到$N_A / (N_A + N_B)=P_A$。所以赔率最终是有市场平均认为的A队胜率决定的。

在博彩公司有抽成的情况下基本结论基本不变。如果市场平均判断水平很低，$N_A/(N_A+N_B)$与真实胜率$P_A$差别很大，区间$[1 - 1/ O_B, 1 / O_A]$覆盖了$N_A/(N_A+N_B)$，但是不覆盖$P_A$。那么投注者就出现了正收益的机会。不妨我们假设$P_A > N_A/(N_A+N_B)$，虽然$1/O_A$会比$N_A/(N_A+N_B)$略大（由博彩公司决定），但是我们再假定$P_A > 1 / O_A$，此时投注A队伍是有正收益的。如果市场平均判断水平很高，$N_A/(N_A+N_B)$与真实胜率$P_A$差别很小，区间$[1 - 1/ O_B, 1 / O_A]$覆盖了$N_A/(N_A+N_B)$，同时覆盖了$P_A$。那么无论投注哪支队伍，收益期望的都是负的，只有博彩公司收益期望是正的，而且是没有风险的。

C++整数系统的复杂性源于设计之初就语焉不详，为了在不同平台上达到最佳性能，没有规定每一种整数的长度。以现在的眼光看来当然是得不偿失，可移植的代码本来就不好写，可移植的性能更是难上加难。所以后来设计的语言中每种整数类型都是固定长度。当然固定整数长度也有缺陷，比如几乎所有语言中都有一种叫做int的类型，他是典型的整数长度，应该足够长，性能足够好，那么现在就有一个问题，应该将int定位32位还是64位？或许根据平台浮动的定义才是最好的。

char

  • 如果其中一个操作子为long double，那么两个操作子都转换为long double。(unsigned) long long转换为long double的时候可能会丢失精度。

C++03中的常量表达式

C++03中就已经有常量表达式了，比如

C++03中const关键字扮演着多种角色

可见const关键字可以定义不可更改变量，也可以定义常量表达式，当对不可更改变量的定义满足一定条件时候，不可更改变量自动提升为常量表达式。

常量表达式的优势

  • 常量表达式在编译期求值
    • 常量表达式可以用于静态长度数组的长度
    • 常量表达式可以用于模板参数
    • 常量表达式可以用于switch的caselabel
  • 常量表达式是编译期概念，所以可以定义在头文件
  • 常量表达式没有初始化顺序问题

lackage

虽然让len具有static的链接性解决了len在不同翻译单元重复定义的问题，是一种简便的解决方案。但是这导致每个翻译单元都一份len的拷贝，同时也破坏了const和static的正交性，实则不是一种优雅的解决方案。

C++11中的常量表达式

constexpr varible

C++11中可以使用constexpr定义constexpr variable,constexpr variable是一种常量表达式

可见constexpr相比const可以保证被定义的对象一定是常量表达式。如果意图定义常量表达式，那么就使用constexpr 这样可以避免编译错误传播。如果意图定义不可更改变量，那么就使用const，可以不管被定义的对象被是否提升为常量表达式。

const只支持定义整数，枚举和指针类型的常量表达式。constexpr除了可以定义build-in类型的常量表达式，还可以定义任何literal-type类型的常量表达式。

  • 如果NRVO没有发生，如果对象有移动构造函数，那么就使用移动构造函数构造上层函数局部对象
  • 如果NRVO和移动构造都没有发生，如果对象有复制构造函数，那么就使用复制构造函数构造上层函数局部对象
